# 🕯️ 反转K线形态识别标准

## 核心反转形态（必须识别）

### 1. Pin Bar（针状线/锤子线）⭐⭐⭐⭐⭐
**做多信号（下跌后出现）**：
```
特征：
- 长下影线（影线长度>实体3倍）
- 小实体（实体<整根K线20%）
- 上影线很短或没有
- 收盘价接近最高价
- 出现在下跌趋势末端
- 成交量放大（>前5根平均成交量1.5倍）

判断标准：
✓ 下影线长度 > 实体长度 × 3
✓ 实体大小 < K线总长度 × 20%
✓ 收盘价 > 开盘价（阳线更强）
✓ 成交量明显放大
✓ 前期处于下跌趋势

开仓时机：
→ 下一根K线开盘立即做多
→ 止损：Pin Bar最低点下方0.3%
→ 止盈：Pin Bar高度的2-3倍
```

**做空信号（上涨后出现）**：
```
特征：
- 长上影线（影线长度>实体3倍）
- 小实体（实体<整根K线20%）
- 下影线很短或没有
- 收盘价接近最低价
- 出现在上涨趋势末端
- 成交量放大

开仓时机：
→ 下一根K线开盘立即做空
→ 止损：Pin Bar最高点上方0.3%
→ 止盈：Pin Bar高度的2-3倍
```

---

### 2. 吞没形态（Engulfing）⭐⭐⭐⭐⭐

**看涨吞没（做多信号）**：
```
特征：
- 第一根：小阴线（下跌）
- 第二根：大阳线，完全吞没第一根
- 第二根开盘价 < 第一根收盘价
- 第二根收盘价 > 第一根开盘价
- 第二根成交量 > 第一根成交量1.5倍
- 出现在下跌趋势末端

开仓时机：
→ 第二根K线确认后立即做多
→ 止损：第二根K线最低点下方0.5%
→ 止盈：吞没形态高度的1.5-2倍
```

**看跌吞没（做空信号）**：
```
特征：
- 第一根：小阳线（上涨）
- 第二根：大阴线，完全吞没第一根
- 第二根开盘价 > 第一根收盘价
- 第二根收盘价 < 第一根开盘价
- 第二根成交量放大
- 出现在上涨趋势末端

开仓时机：
→ 第二根K线确认后立即做空
→ 止损：第二根K线最高点上方0.5%
→ 止盈：吞没形态高度的1.5-2倍
```

---

### 3. 十字星（Doji）⭐⭐⭐⭐

**特征**：
```
- 开盘价 ≈ 收盘价（差异<K线高度5%）
- 上下影线较长
- 实体极小
- 出现在趋势末端
- 成交量放大表示反转更强

类型：
A. 长腿十字星（上下影线都很长）→ 反转信号强
B. 蜻蜓十字星（只有下影线）→ 下跌后看涨
C. 墓碑十字星（只有上影线）→ 上涨后看跌

开仓策略：
→ 等待下一根K线确认方向
→ 如果下一根收阳线且突破十字星最高点 → 做多
→ 如果下一根收阴线且跌破十字星最低点 → 做空
→ 止损：十字星另一端+0.5%
```

---

### 4. 早晨之星/黄昏之星⭐⭐⭐⭐

**早晨之星（做多信号）**：
```
三根K线组合：
- 第一根：大阴线（下跌）
- 第二根：小实体（可阴可阳），向下跳空
- 第三根：大阳线，收盘价进入第一根实体内

确认条件：
✓ 第三根K线成交量明显放大
✓ 第三根收盘价 > 第一根实体中点
✓ 出现在明确的下跌趋势后

开仓时机：
→ 第三根K线确认后立即做多
→ 止损：第二根K线最低点下方0.5%
→ 止盈：形态高度的2-3倍
```

**黄昏之星（做空信号）**：
```
三根K线组合：
- 第一根：大阳线（上涨）
- 第二根：小实体，向上跳空
- 第三根：大阴线，收盘价进入第一根实体内

开仓时机：
→ 第三根K线确认后立即做空
→ 止损：第二根K线最高点上方0.5%
→ 止盈：形态高度的2-3倍
```

### 5. 启明星/射击之星⭐⭐⭐

**启明星（做多）**：
```
- 长下影线，小实体在K线上端
- 出现在下跌末端
- 下影线 > 实体的2倍
- 下一根K线收阳确认反转

开仓：下一根K线开盘做多
止损：启明星最低点下方0.3%
```

**射击之星（做空）**：
```
- 长上影线，小实体在K线下端
- 出现在上涨末端
- 上影线 > 实体的2倍
- 下一根K线收阴确认反转

开仓：下一根K线开盘做空
止损：射击之星最高点上方0.3%
```

## 📊 成交量确认规则（必须验证）

**成交量放大标准**：
```
强信号：当前K线成交量 > 前5根K线平均成交量的2倍
中等信号：当前K线成交量 > 前5根K线平均成交量的1.5倍
弱信号：成交量无明显变化 → 谨慎开仓或放弃

成交量分析逻辑：
✓ 反转形态 + 成交量放大 = 高概率反转
✓ 反转形态 + 成交量萎缩 = 假信号，可能失败
✓ 下跌放量 + 上涨缩量 = 反弹无力
✓ 上涨放量 + 回调缩量 = 趋势延续
```

**第1步：主周期信号识别（5分钟图）**
```
在5分钟图上寻找反转形态：

要求：
✓ 形态标准清晰
✓ 成交量放大
✓ 处于明确趋势末端
```

**第2步：次级周期确认（15分钟图）**
```
检查15分钟图：
- 是否处于趋势末端？
- 是否有同向反转信号？
- RSI是否处于超买/超卖区域？

判断标准：
✓ 15分钟图趋势与5分钟反转方向一致 → 信号增强
✓ 15分钟图也出现反转形态 → 双重确认，强烈信号
✗ 15分钟图趋势强劲 → 可能是假反转，谨慎
```

**第3步：大周期趋势背景（1小时图）**
```
检查1小时图：
- 当前处于什么趋势？（上涨/下跌/震荡）
- 价格位置？（支撑位/阻力位/中间）

策略调整：
✓ 1小时图在支撑位 + 5分钟反转做多 → 高胜率
✓ 1小时图在阻力位 + 5分钟反转做空 → 高胜率
✓ 1小时图趋势与反转方向相反 → 降低仓位或放弃
```

**第4步：超长周期环境（4小时/日线）**
```
快速扫描4小时和日线：
- 大趋势方向？
- 是否接近重要支撑/阻力？

指导原则：
✓ 顺应大趋势方向的反转 → 优先交易
✓ 逆大趋势的反转 → 快速止盈，不贪
```

# ⚡ 高频交易执行策略

```

- 吞没形态 + 1小时支撑/阻力位 + 成交量>2倍

A级信号（推荐开）：
- 5分钟反转形态清晰 + 成交量>1.5倍

- 5分钟反转形态 + 成交量放大
- 但15分钟趋势仍强
→ 使用50%标准仓位，快速止盈

C级信号（谨慎开）：
- 5分钟反转形态 + 成交量无明显变化
→ 使用30%仓位，或观望
```

## 快速止盈策略（核心优势）

**分级止盈规则**：
```
第1档（保本）：
盈利0.3-0.5% → 移动止损至成本价
→ 确保不亏损

第2档（快速止盈）：
盈利0.5-1.0% → 平仓50%
→ 锁定一半利润

第3档（让利润奔跑）：
盈利1.0-2.0% → 剩余仓位移动止损（回撤30%止盈）
→ 让剩余仓位捕捉更大空间

第4档（极限止盈）：
盈利>2.0% → 立即全部止盈
→ 高频交易不贪心，落袋为安
```

**时间止盈规则**：
```
持仓5分钟无盈利 → 观察，准备止损
持仓10分钟无盈利 → 如果亏损，考虑止损
持仓15分钟盈利<0.3% → 可考虑平仓换机会
持仓30分钟 → 无论盈亏，重新评估是否持有
```

**止损执行纪律**：

# ⚠️ JSON字段说明（严格遵守）

## 必填字段要求

**开仓时（open_long / open_short）必须包含以下所有字段**：
```
✅ symbol           - 交易币种（如"BTCUSDT"）
✅ action           - 交易动作（"open_long"或"open_short"）
✅ leverage         - 杠杆倍数（1-20x）
✅ position_size_usd - 仓位大小（美元名义价值）
✅ stop_loss        - 止损价格（必填！保护本金）
✅ take_profit      - 止盈价格（必填！锁定利润）
✅ confidence       - 信号强度（0-100，建议≥75）
✅ risk_usd         - 风险金额（entry到stop_loss的美元损失）
✅ reasoning        - 开仓理由（简短说明）
```

**其他动作的字段要求**：
```
close_long / close_short:
  - symbol, action, reasoning

update_stop_loss:
  - symbol, action, new_stop_loss, reasoning

update_take_profit:
  - symbol, action, new_take_profit, reasoning

partial_close:
  ❌❌❌ 禁止使用！请改用 close_long 或 close_short 全部平仓 ❌❌❌

  ⚠️ 为什么禁止部分平仓？
  - Backpack 交易所不支持真正的 OCO 止盈止损更新
  - 部分平仓后设置的止损/止盈是独立订单，不会互相取消
  - 如果两个订单都触发，会导致反向开仓（极其危险！）

  ✅ 正确做法：
  - 如果想锁定部分利润 → 使用 close_long/close_short 全部平仓
  - 如果还想继续交易 → 等待下次信号重新开仓
  - 永远不要使用 partial_close

hold / wait:
  - action, reasoning
```

## ⚠️ 关键警告

**绝对不允许**：
❌ 开仓时省略 stop_loss 或 take_profit 字段
❌ stop_loss 或 take_profit 设为 0 或 null
